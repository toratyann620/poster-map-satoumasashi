# Antigravity エージェント引き継ぎ用ドキュメント (AGENT_HANDOVER.md)

このドキュメントは、別のPCで Antigravity IDE を利用して本プロジェクトの開発を引き継ぐ際、新しいPC側のAIエージェントにこれまでの経緯と現在のステータスを即座に把握させるためのものです。

新PCで Antigravity と対話を開始する際は、**「ルート直下の AGENT_HANDOVER.md を読み、これまでの開発経緯と現在の課題を把握してください」** と指示してください。

---

## 1. プロジェクト基本情報
- **アプリケーション名**: ポスターマップ管理システム
- **フロントエンド**: React 19 / TypeScript / Vite 7 / Tailwind CSS 4
- **バックエンド/BaaS**: Firebase (v12)
  - **Authentication**: 管理者およびスタッフのログイン認証
  - **Firestore**: ポスターデータ (`posters`)、変更履歴 (`activityLogs`)、ユーザーデータ (`users`)
  - **Storage**: ポスターの写真アップロード
- **地図サービス**: Google Maps Platform (Maps JavaScript API - AdvancedMarkerElement / Places API / Geocoding API)
- **デプロイ先**: Vercel

---

## 2. これまでの主な修正・変更履歴

### ① Firebase Storage 画像アップロード不具合の修正 (解決済み)
- **問題**: アプリケーションからポスターの写真をアップロードする際、「画像のアップロードに失敗しました」と表示されて登録できない不具合が発生していた。
- **原因**: 
  - 画像データをBase64形式のテキスト（`uploadString`）としてアップロードしていたため、大容量の写真のアップロードでエラーになりやすかった。
  - ポスター新規追加時にドキュメントIDが決まっておらず、Storageのパス名が衝突・権限エラーを引き起こしていた。
- **対応**: 
  - `PinBottomSheet.tsx` 内で `imageCompression` による圧縮後、Base64への変換を廃止し、Blob（ファイル形式）のまま直接 `uploadBytes` を用いてアップロードする実装に変更。
  - 新規作成時のパス名衝突を回避するため、`new_entry` または一時IDフォルダを自動生成して保存するようにパスロジックを変更。

### ② 新機能：変更履歴ダッシュボードの実装 (解決済み)
- **問題 / 要望**: 実際にいつ、誰が、どこを、どう変更したのかの履歴を管理者が把握したい。
- **対応**:
  - Firestoreに `activityLogs` コレクションを新設。
  - `usePosterData.ts` の `addPoster`, `updatePoster`, `deletePoster` が実行されるたび、操作者・日時・変更内容の差分（例:「ステータス: 未設置→設置済」）をログとして保存するロジックを実装。
  - `AdminPanel.tsx` に「ユーザー管理」と「変更履歴」のタブ切り替えを追加し、管理者権限のユーザーのみが履歴リストを閲覧できるようにダッシュボードを新設。

### ③ 新機能：ポスター枚数ウィジェットの実装 (解決済み)
- **問題 / 要望**: 現在のポスター総枚数と、先週比で何枚増えたかを全ユーザーがマップ上で確認できるようにしたい。
- **対応**:
  - `PosterCountWidget.tsx` コンポーネントを新規作成。
  - 過去7日以内に作成されたポスター数を算出して先週比（例:「先週比 +5枚」）を表示するウィジェットをマップ右下に常時表示。

## 3. 現在残っている課題と対応方法

### 🚨 Vercel本番環境でのみ、Google Maps背景タイルがグレーアウトする問題 (未解決)
- **現象**: Vercelにデプロイされた本番環境（ `https://poster-map-app.vercel.app/` ）において、マップのマーカーや検索UIは表示されるが、背後の道路や地形などのタイル画像がロードされず、グレーアウトして「この地域の詳細画像は表示できません」と表示される。
- **切り分け済みの事実**:
  - **ローカル環境 (`localhost:5173`) では完全かつ正常に表示される**（同じAPIキー、Map IDを使用）。
  - Vite/Reactアプリケーションを介さない、APIキーとMap IDを直書きしただけのプレーンなHTMLファイル（ `public/map-test.html` ）をVercelに配置してアクセスしても、全く同様にCORSエラーとなりグレーアウトする。
  - ブラウザのコンソールには、Google Mapsのタイルリクエスト (`https://maps.googleapis.com/maps/vt/pb=...`) に対するCORSエラーや `net::ERR_BLOCKED_BY_ORB` が多発している。
  - これより、フロントエンドコードのバグではなく、**Google Cloud Console（GCP）側のAPIキー制限設定（リファラーのドメイン承認）**あるいは課金ステータスのドメイン制限に起因していると判断。
- **試した対応**:
  - `index.html` に `<meta name="referrer" content="no-referrer-when-downgrade" />` を追加してリファラーが送信されるように調整。
- **次のステップ**:
  - 外部エンジニア等による、GCP管理画面側の「APIキーのウェブサイト（リファラー）制限」の詳細設定見直し（ワイルドカードの指定形式 `*poster-map-app.vercel.app/*` 等が正しく反映されているかの確認）。

## 4. 開発環境のセットアップ方法 (新PCでの手順)
1. GitHubからリポジトリをクローンする。
2. 前のPCから `.env.local` ファイルを安全な方法（チャット等）でコピーし、本プロジェクトのルートディレクトリに配置する。
3. `npm install` を実行し、依存パッケージをインストールする。
4. `npm run dev` でローカルサーバーを起動し、ブラウザで動作確認を行う。
5. デプロイは `npx vercel --prod` を使用（要Vercelアカウントのログイン）。
